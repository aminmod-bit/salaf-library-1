# Salaf Library — Master Plan 2026–2030+

Документ фиксирует полное видение развития Salaf Library как большой исламской цифровой библиотеки на долгие годы.

## 0. Главный принцип

Salaf Library — не просто сайт с PDF. Это исламская цифровая экосистема знаний:

- книги;
- статьи;
- азкары и дуа;
- хадисы;
- биографии;
- Коран в будущем;
- аудио в будущем;
- поиск;
- языковые разделы;
- PWA;
- мобильное приложение;
- админка;
- SEO;
- Cloudflare-инфраструктура.

Проект должен быть:

- надёжным;
- красивым;
- быстрым;
- масштабируемым;
- понятным;
- удобным на телефоне, планшете и компьютере;
- подготовленным к долгосрочной эксплуатации.

Архитектуру без необходимости не ломать. Существующую структуру сохранять и развивать постепенно.

## 3. Книги

### 3.1. Папочная структура

Основная структура:

```text
Books/
  Aqeedah/
  Tawhid/
  Manhaj/
  Tafsir/
  Hadith/
  Seerah/
  Fiqh/
  Arabic/
  Dawah/
  History/
  Biography/
  Children/
  Azkar/
  Dua/
  Other/
```

В будущем языковая структура:

```text
Books/ru/Aqeedah/
Books/tj/Aqeedah/
Books/uz/Fiqh/
Books/fa/Tafsir/
Books/ar/Hadith/
Books/en/Dawah/
```

### 3.2. Автоматический импорт

Идеальный процесс:

1. пользователь выбирает PDF;
2. система читает PDF;
3. определяет название;
4. определяет автора;
5. определяет категорию;
6. определяет язык;
7. определяет страницы;
8. определяет ISBN, издательство, переводчика, редактора;
9. создаёт slug;
10. создаёт cover.webp и thumb.webp;
11. создаёт JSON;
12. кладёт PDF в правильную папку;
13. обновляет `books.json`;
14. обновляет search index;
15. обновляет sitemap и RSS;
16. показывает отчёт.

### 3.3. Качество метаданных

Админка должна показывать:

- автор не указан;
- категория «Другое»;
- нет описания;
- нет PDF;
- нет языка;
- нет тегов;
- нет обложки;
- PDF не открывается.
