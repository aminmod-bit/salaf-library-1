# Руководство администратора контента Salaf Library

## Быстрый старт

### Добавление книг

```bash
# 1. Положите PDF-файлы в import/books/inbox/
#    cp /путь/к/книге.pdf import/books/inbox/

# 2. Сканируйте PDF
npm run content:scan

# 3. Проверьте черновики в админке
#    http://localhost:5173/#/admin/content

# 4. Импортируйте одобренные черновики
npm run content:import

# 5. Проверьте валидность
npm run content:validate
```

### Структура проекта

```
salaf-library-1/
├── public/data/           # JSON-данные (books.json, azkar.json и др.)
├── public/books/          # PDF-файлы книг
├── public/covers/         # Обложки книг
├── import/
│   └── books/
│       ├── inbox/         # Кладёте PDF сюда
│       └── drafts/        # Черновики после сканирования
├── backups/               # Автоматические бэкапы
├── scripts/content.mjs    # Скрипт управления контентом
└── src/pages/admin/
    └── AdminContentPage.tsx  # Админка контента
```

## Добавление книг

### Автоматический импорт

1. Положите PDF в `import/books/inbox/`
2. Запустите `npm run content:scan`
3. Скрипт автоматически определит:
   - Название (из имени файла)
   - Автора (из имени файла, паттерн "Автор - Название")
   - Категорию (по ключевым словам)
   - Теги (по ключевым словам)
   - Язык (по имени файла)
   - Размер файла

4. Если что-то определить не удалось — стоит `needsReview: true`
5. Проверьте черновики в админке `#/admin/content`
6. Нажмите "Опубликовать" для готовых черновиков

### Ручное добавление

1. Откройте `#/admin/content`
2. Нажмите "Импорт JSON"
3. Выберите JSON-файл со структурой:
```json
[
  {
    "title": "Название книги",
    "author": "Автор",
    "category": "Акыда",
    "language": "Русский",
    "description": "Описание",
    "tags": ["тег1", "тег2"]
  }
]
```

### Импорт через админку

1. Откройте `#/admin/import`
2. Перетащите PDF-файлы или нажмите "Выбрать PDF"
3. Система проанализирует каждую книгу
4. Проверьте результаты
5. Нажмите "Импортировать автоматически"

## Редактирование книг

### Через админку

1. Откройте `#/admin/content`
2. Найдите книгу (поиск или фильтр)
3. Нажмите иконку редактирования (карандаш)
4. Измените нужные поля:
   - Название
   - Автор
   - Категория
   - Теги (через запятую)
   - Описание
   - Язык
   - Год
   - Обложка (путь: `./covers/name.webp`)
   - Рекомендуемая / Новинка / Популярная
5. Нажмите "Сохранить"

### Через JSON

1. Скачайте books.json из админки
2. Отредактируйте в текстовом редакторе
3. Замените `public/data/books.json`
4. Запустите `npm run build`

## Удаление книг

### Мягкое удаление (корзина)

1. Откройте `#/admin/content`
2. Найдите книгу
3. Нажмите иконку корзины
4. Подтвердите удаление
5. Книга переместится в корзину

### Восстановление

1. Перейдите на вкладку "Корзина"
2. Нажмите "Восстановить"

### Окончательное удаление

1. Перейдите на вкладку "Корзина"
2. Нажмите иконку корзины (красная)
3. Подтвердите — книга удалена навсегда

## Бэкапы

### Автоматический бэкап

```bash
npm run content:backup
```

Сохраняет все JSON-файлы в `backups/` с временной меткой.

### Восстановление

```bash
npm run content:restore
```

Восстанавливает данные из последнего бэкапа.

### Бэкап перед изменениями

Скрипт `content:import` автоматически создаёт бэкап перед импортом.

## Обложки

### Форматы

- WebP (рекомендуется)
- PNG
- JPG/JPEG

### Размеры

- Обложка: 300×400px (3:4)
- Превью: 150×200px

### Путь

Указывается относительно корня сайта:
```
./covers/book-name.webp
```

### Автоматическое создание

При импорте PDF скрипт пытается создать обложку из первой страницы.
Если не удалось — используется заглушка с эмодзи.

## Валидация

```bash
npm run content:validate
```

Проверяет:
- Все JSON-файлы валидны
- Нет пустых ID
- Нет дублей ID
- Есть title/name
- Есть category (для книг)
- tags — массив

## Частые ошибки

### "No PDF files found"

Файлы не в `import/books/inbox/`. Проверьте путь.

### "Invalid JSON"

Файл books.json повреждён. Восстановите из бэкапа:
```bash
npm run content:restore
```

### Книга не отображается

1. Проверьте `npm run content:validate`
2. Убедитесь что книга есть в `public/data/books.json`
3. Пересоберите `npm run build`

### Обложка не загружается

1. Проверьте путь к файлу
2. Убедитесь что файл в `public/covers/`
3. Путь должен быть `./covers/name.webp`
